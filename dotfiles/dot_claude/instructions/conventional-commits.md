## Commit messages

Use Angular-style conventional commit messages: `type: description`
- Types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`
- For breaking changes, add `!` after the type (e.g. `feat!: remove deprecated API`) and include a `BREAKING CHANGE: <description>` footer in the commit body; for multiple breaking changes, add one `BREAKING CHANGE:` footer per change
